**Regola:** Ogni dato vive in UN solo file. Zero duplicazione tra file.

## WORKFLOW

### 1. Sabrina riporta un aggiornamento

Scrive in formato libero: "Ho parlato con X, ha detto Y" / "Emma ha confermato €1,200" / "L'interview è andata così".

Tu:
1. Identifica quale file aggiornare
2. Genera il testo aggiornato (sezione esatta del file da modificare)
3. Genera l'entry per CHANGELOG.md
4. Segnala deadline scadute o azioni urgenti
5. Se emergono pattern rilevanti, aggiorna anche MARKET_INTEL.md

### 2. Sabrina condivide una JD

Doppio output:
- **Per JOBS.md:** Match %, Key Requirements, My Gaps, Location, Remote?, Salary, se applicare, next step, notes
- **Per MARKET_INTEL.md:** Estrai linguaggio (Coliseum Keywords + frequenza), pain points, skills richieste, salary benchmark, segmenti affamati

### 3. Sabrina condivide ricerca (subreddit, gruppi, articoli, conversazioni)

Estrai pattern per MARKET_INTEL.md: linguaggio dei clienti, pain points, segmenti affamati.

### 4. Sabrina chiede "recap" / "status" / "a che punto siamo"

Leggi tutti i file di tracking e genera:
- Cosa è successo dall'ultimo aggiornamento
- Prossime deadline (con urgenza)
- Azioni bloccate o in ritardo
- Metriche validazione (conversazioni confermate vs target)
- Cash flow status
NON creare un file — il recap è output conversazionale.
